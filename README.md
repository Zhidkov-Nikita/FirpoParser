# Парсер ИРПО на Django + Playwright

### О проекте
Этот проект представляет собой автоматизированный синхронный скрапер для платформы ИРПО (`edu.firpo.ru`), интегрированный с веб-интерфейсом на базе **Django**. Проект полностью очищен от старого фронтенда: единственным интерфейсом управления является встроенная **Django-админка**, которая сразу открывается при переходе на сайт.

Парсер полностью имитирует действия реального человека, обходит защиту авторизации, скачивает файлы документов напрямую через сессию браузера и раскладывает данные по связанным таблицам базы данных.

---

## Архитектура базы данных (Django)

Вся база данных построена на строгой реляционной схеме. Информация разделена на 6 логических таблиц:

1. **Обучаемый (`Student`)** — базовая карточка студента. Хранит ФИО, уникальный ID из системы, почту, а также физические PDF-файлы документов, скачанные парсером.
2. **5 связанных таблиц документов (Связь 1-к-1 со Студентом):**
   * Поступление на курс (`CourseEnrollment`)
   * Согласие на обработку ПДн (`PersonalDataConsent`)
   * Заявление на зачисление (`EnrollmentApplication`)
   * Договор на обучение (`EducationContract`)
   * Первичные документы (`PrimaryDocuments`)

*Каждая из 5 таблиц документов содержит статус проверки (галочка True/False), дату операции и имя оператора.*

## Алгоритм работы скрапера (Человеческим языком)

Скрипт работает в **строго синхронном режиме** (без async/await), что гарантирует 100% стабильность работы базы данных Django. Логика шагов выглядит так:

1. **Авторизация**: Парсер берет данные из `scraper_config.yaml`, заходит на страницу входа и заполняет поля.
2. **Обход лимитов**: На странице студентов парсер подменяет ключ `preset-students30-students30-all-last` в `localStorage` браузера на значение `1184` и обновляет страницу. Это заставляет интерфейс Quasar сразу показать полный список студентов.
3. **Сбор Email**: Скрипт собирает список уникальных email-адресов со страницы.
4. **Цикл по студентам**: 
   * Playwright находит ячейку `<td>` с конкретным Email и кликает по ней.
   * **Ожидание (Polling)**: Скрипт дает системе до 5 попыток (с паузой 2 сек), чтобы модальное окно (диалог) Quasar полностью загрузилось.
   * **Скачивание PDF**: Скрипт ищет иконки открытия документов, перехватывает открытие новой вкладки и скачивает байты PDF-файла через встроенный JavaScript-запрос `fetch` (чтобы сервер не сбросил скачивание из-за отсутствия авторизации). Файлы сохраняются в Django.
   * **Вкладка «Маршрут»**: Парсер кликает на второй таб модального окна, дожидается появления таблицы «Маршрут» и считывает первые 5 строк. По текстовому названию строки скрипт автоматически понимает, в какую из 5 таблиц документов нужно сохранить статус-галочку, дату и оператора.
   * **Очистка сессии**: Нажимается клавиша `Escape`, диалог закрывается, а скрипт динамически ждет, пока основная таблица студентов (`tbody.q-virtual-scroll__content`) снова станет видимой, после чего идет к следующему email.

## Требования к окружению и настройка

Так как проект запускается на **Ubuntu 26.04**, где встроенные бинарники Playwright не поддерживаются, парсер настроен на использование **системного Chromium**.

### 1. Установка системного браузера в Ubuntu:
```bash
sudo apt update
sudo apt install -y chromium-browser
```

### 2. Установка зависимостей (внутри venv):
```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
# Примечание: команду "playwright install" запускать НЕ нужно!
```

### 3. Конфигурация:
* Переименуйте файл `scraper_config.example.yaml` in `scraper_config.yaml` в корне проекта и впишите туда ваши доступы от платформы.
* Убедитесь, что в `settings.py` раскомментирован `DEBUG = True`, а параметр `ALLOWED_HOSTS = ['*']` (для исключения ошибок 400 в OpenCode).

## Команды для запуска

Перед запуском парсера обязательно примените миграции базы данных и создайте администратора:
```bash
python manage.py makemigrations
python manage.py migrate
python manage.py createsuperuser
```

### Запуск скрапера в режиме ТЕСТА (только первые 10 студентов):
Используйте эту команду, чтобы быстро проверить корректность работы селекторов и логирования:
```bash
python manage.py run_scraper --test
```

### Запуск скрапера в ОБЫЧНОМ режиме (полный обход базы):
```bash
python manage.py run_scraper
```

*Во время работы скрипт выводит подробный `[DEBUG]` лог по каждой строке, ячейке и операции сохранения, что позволяет сразу увидеть любые изменения в верстке платформы ИРПО.*
